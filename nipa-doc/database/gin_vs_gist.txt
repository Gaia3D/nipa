gin 인덱스 조회는 gist 보다 3배 정도 빠르다.
gin 인덱스는 구축하는데 gist보다 3배 정도 오래 걸린다.
gin 인덱스는 gist 인덱스보다 업데이트가 약간 느리지만 빠른 업데이트 지원이 비활성화 되면 10배 정도 느려진다.
gin 인덱스는 gist 인덱스보다 2~3배 크다.

정적 인덱스는 gin가 최선
동적인 경우 gist 인덱스는 업데이트가 더 빠르다.

gist 인덱스는 동적 데이터의 경우 매우 우수하고, 고유어(어휘소)가 100, 000개 이하일때는 빠른 반면
gin 인덱스는 100, 000 개 이상의 어휘소 처리할 때 훨씬 낫지만 업데이트가 느리다.

이걸 사용해야 하는거 같음
create index 인덱스명 on 테이블 using gin(tsvector(컬럼));
tsquery 등을 고민해 봐야 함
